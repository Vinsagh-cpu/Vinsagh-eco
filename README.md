# Vinsagh Eco

Base tecnica y documental de Vinsagh Eco Studios para PACKAGE-001.

## Estado

- Proyecto: Vinsagh Eco Studios
- Package activo: PACKAGE-001
- Orden de trabajo: CODEX-RUN-001
- Modo: Produccion
- Fecha de preparacion: 2026-07-03

## Alcance de esta base

Esta preparacion deja listo el repositorio para continuar con SB-001 o GAME-001 despues de aprobacion humana.

Incluye:

- Estructura documental base en `docs/`.
- Estructura tecnica reservada para Flutter, Flame y packages internos.
- Estructura vacia de assets.
- Manifiesto visual inicial sin assets registrados.
- Reglas minimas de nomenclatura y control tecnico.

No incluye:

- Pantallas finales.
- Login.
- Tienda.
- Founder Club completo.
- Closet completo.
- Firebase activo.
- Assets reales.
- Narrativa, personajes o nombres oficiales nuevos.

## Rutas principales

- `docs/00_canon/`: reservado para canon fundacional. No modificar sin aprobacion humana explicita.
- `docs/01_architecture/`: arquitectura, estructura real e inicializacion tecnica.
- `docs/02_experience/`: puente formativo y criterios de experiencia sin pantallas finales.
- `docs/03_storyboard/`: reservado para storyboard aprobado.
- `docs/04_design_system/`: sistema de diseno, tokens, nomenclatura y assets.
- `apps/mobile_flutter/`: reservado para la app Flutter cuando el entorno lo permita.
- `packages/`: packages internos con prefijo `ve_`.
- `assets/`: biblioteca vacia de carpetas para assets aprobados.
- `firebase/`: reservado; no contiene configuracion activa.
- `tools/`: utilidades futuras del repositorio.

## Regla de oro

Vinsagh Eco se trata como universo narrativo interactivo. La implementacion tecnica debe proteger el canon, evitar assets improvisados y mantener cada cambio revisable.
