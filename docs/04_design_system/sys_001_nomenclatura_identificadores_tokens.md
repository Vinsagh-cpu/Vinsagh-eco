# SYS-001 - Nomenclatura, identificadores y tokens

## Nomenclatura

- Carpetas y archivos: `snake_case`.
- Packages internos: prefijo `ve_`.
- Identificadores de sistema: prefijo `VE`.
- Documentos controlados: prefijo del entregable, por ejemplo `ARCH-001`.

## Identificadores sugeridos por tipo

- Assets: `VE-ASSET-<categoria>-<numero>`
- Componentes UI: `VE-UI-<componente>-<numero>`
- Escenas tecnicas: `VE-SCENE-<nombre>-<numero>`
- Eventos: `VE-EVENT-<nombre>-<numero>`

## Estados

- `draft`: propuesto, no aprobado.
- `review`: listo para revision humana.
- `approved`: aprobado para uso.
- `deprecated`: retirado o reemplazado.

## Tokens

DS-001 aun no aprueba valores visuales finales. Los tokens pueden definirse por nombre antes de tener valores definitivos, pero no deben inventar paleta, tipografia o estilo final.
