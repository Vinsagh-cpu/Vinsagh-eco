# MVP-001 - In Scope

## Proposito

Precisar que si entra en la primera version funcional minima.

## Incluido

### Experiencia inicial

- Primera sesion del Agente.
- Activacion inicial.
- Umbral de entrada.
- Bienvenida.
- Primera eleccion.
- Microaccion de intencion.
- Primer desbloqueo simbolico.
- Vista de continuidad.

### Documentacion base

- UX-001 documentado.
- SB-001 documentado.
- GAME-001 documentado.
- FIRE-001 documentado.
- SYS-001 documentado.
- DS-001 documentado.
- ART-001 documentado.

### Arquitectura de referencia

- Estados reservados del Agente.
- Pantallas conceptuales.
- Escenas Flame reservadas.
- Eventos UX, GAME y backend reservados.
- Colecciones conceptuales de Firebase.
- Modelo base del Agente.
- Limites de seguridad conceptuales.

### Preparacion del repositorio

- Estructura documental ordenada.
- Manifiesto visual preparado.
- Nomenclatura tecnica consistente con `VE`.
- Assets solo como referencias documentadas.

## Lectura funcional minima

El MVP debe permitir explicar con claridad:

- Que vive el Agente en su primera entrada.
- Que momentos conceptuales existen.
- Que eventos se reservan.
- Que datos deberan guardarse despues.
- Que escenas Flame se usaran posteriormente.
- Que queda fuera de esta primera version.
- Que necesita estar listo antes de programar.

## Criterio de inclusion

Todo lo que dependa de implementacion avanzada, assets finales o backend activo queda fuera de este scope minimo, aunque pertenezca al producto a largo plazo.
