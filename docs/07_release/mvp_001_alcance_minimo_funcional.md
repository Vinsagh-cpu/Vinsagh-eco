# MVP-001 - Alcance minimo de primera version funcional

## Estado

Documento tecnico inicial. Sin codigo, sin pantallas Flutter y sin Firebase activo.

## Identificacion

- Entregable: MVP-001
- Orden de trabajo: CODEX-RUN-006
- Package activo: PACKAGE-001
- Base documental: UX-001, SB-001, GAME-001 y FIRE-001

## Proposito

Definir el alcance minimo funcional de la primera version de Vinsagh Eco Studios para validar la experiencia inicial del Agente sin construir todavia todo el universo.

## Principio rector

El MVP debe demostrar la entrada al universo, no la totalidad del universo.

## Que entra en MVP-001

- Primera sesion del Agente documentada.
- Flujo de activacion inicial.
- Estados del Agente reservados.
- Primera eleccion del Agente.
- Microaccion inicial.
- Primer desbloqueo simbolico.
- Storyboard de primera sesion.
- Mapa conceptual de escenas Flame.
- Diseno base de Firebase documental.
- Sistema de nombres, tokens y assets documentado.
- Manifiesto visual preparado.
- Repositorio ordenado.

## Que no entra en MVP-001

- Login real.
- Firebase activo.
- Auth.
- Firestore real.
- Tienda.
- Pagos.
- Founder Club completo.
- Closet.
- Red social.
- Chat.
- Personajes completos de Vin y Sagh.
- Assets finales.
- Animaciones finales.
- Publicacion en Play Store.
- Metricas reales.
- Panel administrativo.

## Lectura de alcance

MVP-001 no cierra el universo. Solo valida que la entrada funciona como experiencia, que los documentos tecnicos ya estan conectados y que la primera version futura tendra una base segura para programarse.

## Dependencias documentales

- UX-001 define la experiencia emocional.
- SB-001 ordena el recorrido y los beats.
- GAME-001 reserva la capa interactiva y las escenas Flame.
- FIRE-001 define la capa documental de datos.

## Riesgos controlados

- Confundir entrada al universo con producto completo.
- Acelerar hacia pantallas finales antes de validar la base documental.
- Mezclar backend documental con implementacion real.
- Introducir assets o personajes que aun no tienen aprobacion.

## Resultado esperado

MVP-001 deja el paquete listo para cerrar PACKAGE-001 documental y preparar una futura fase de implementacion tecnica controlada.
