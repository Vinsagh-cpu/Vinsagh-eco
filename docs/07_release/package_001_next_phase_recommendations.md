# PACKAGE-001 - Next Phase Recommendations

## Proposito

Definir la ruta recomendada para la siguiente fase despues del cierre documental de PACKAGE-001.

## Direccion general

PACKAGE-002 debe enfocarse en implementacion tecnica controlada y aprendizaje guiado.

La siguiente fase no debe empezar con features complejas. Debe empezar con la minima implementacion segura que permita validar Flutter, estructura, tokens, una pantalla tecnica y una primera escena interactiva reducida.

## Ruta recomendada

1. `TRAIN-004` - Flutter practico aplicado a este repositorio.
2. `INIT-002` - Inicializacion real Flutter cuando el entorno este listo.
3. `UIKIT-001` - Implementacion inicial de tokens visuales.
4. `APP-001` - Pantalla tecnica controlada.
5. `GAME-002` - Primer prototipo Flame minimo.
6. `FIRE-002` - Preparacion no destructiva de Firebase.

## Recomendacion de secuencia

### 1. Aprendizaje aplicado

Primero, reforzar Flutter con ejercicios alineados al repo para que la fundadora pueda leer y mantener la implementacion.

### 2. Inicializacion tecnica real

Despues, crear o activar la base Flutter solo cuando el entorno y la capacidad operativa lo permitan.

### 3. Tokens visuales

Luego, traducir el sistema de diseno a una primera capa utilizable sin abrir UI final.

### 4. Pantalla tecnica

Despues, validar una pantalla de arranque controlada que no represente producto final.

### 5. Prototipo Flame minimo

Luego, probar una escena interactiva reducida para verificar que el puente documental funciona.

### 6. Firebase no destructivo

Al final, preparar Firebase de manera segura, sin activar todavia reglas productivas ni flujos sensibles.

## Riesgos a evitar

- Avanzar a codigo sin dominar Flutter.
- Mezclar FlutterFlow con Flutter sin puente claro.
- Implementar Firebase antes de definir auth.
- Crear assets finales sin manifiesto aprobado.
- Dejar que Codex modifique demasiadas capas a la vez.

## Criterio de exito para PACKAGE-002

PACKAGE-002 debe demostrar una implementacion minima, guiada y validada paso a paso, sin saltar directo a features grandes.

## Decision de ruta

La siguiente fase no debe intentar construir el universo completo.

Debe construir confianza tecnica, una base ejecutable pequeña y una trayectoria de aprendizaje que mantenga el proyecto estable.
