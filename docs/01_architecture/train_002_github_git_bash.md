# TRAIN-002 - GitHub y Git Bash

## Objetivo

Definir una practica minima para versionar PACKAGE-001 sin commits automaticos.

## Flujo recomendado

```powershell
git status --short
git diff
git add <archivo>
git commit -m "Preparar base tecnica PACKAGE-001"
```

## Reglas

- Revisar cambios antes de staging.
- No hacer commit automatico desde Codex sin aprobacion humana.
- No mezclar limpieza de contenido heredado con estructura base sin una orden separada.
- Mantener nombres de archivo en `snake_case`.
